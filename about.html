Project Highlights

Our group’s web application stands out for its simplicity, functionality, and user-centered design. One of the main highlights is the event management platform, which allows users to browse upcoming events with ease. Features such as search, sorting, category-based filtering, and pagination make the browsing experience efficient and enjoyable. The platform ensures that users can quickly find events relevant to their interests without being overwhelmed.

Another key highlight is the implementation of an authentication system entirely on the front end using LocalStorage. This gives users the ability to sign up, log in, and manage their own event registrations. By tying registrations to individual accounts, the application personalizes the experience while demonstrating how user sessions can be managed without a backend.

The design of the platform is also a notable achievement. By leveraging Bootstrap, the interface is responsive and visually consistent across devices. Cards, forms, and navigation elements were structured in a way that balances clarity and accessibility. This helped create a polished look and feel while keeping the layout adaptable.

Finally, the highlight of the project was the teamwork itself. Each member contributed distinct skills, from coding and UI design to content development and documentation. This collaborative approach not only strengthened the final product but also ensured that the workload was balanced and aligned with individual strengths.

Challenges Faced

One of the main challenges our group encountered was integrating multiple features into a single cohesive system. Since our project involved search, sorting, filtering, pagination, and user authentication, it was important to ensure these features worked together smoothly. Coordinating the data flow between JSON files, LocalStorage, and dynamic rendering in JavaScript required careful planning and debugging.

Another challenge was working with LocalStorage for authentication and event registrations. Without a backend, we had to simulate session handling, user-specific data, and registration logic on the client side. This made it tricky to prevent issues such as data overwriting or inconsistent state across different pages. Testing and refining this part of the project took more time than expected but gave us a deeper understanding of how front-end session management can work.

The team also faced difficulties with Bootstrap styling and layout. Making the platform responsive while ensuring it remained clean and accessible required adjustments to the grid system and component design. For example, balancing card content, images, and action buttons in a way that looked good on both desktop and mobile devices was not straightforward.

Lastly, group coordination was itself a challenge. Each member had different strengths and availability, which sometimes made it difficult to synchronize progress. However, by dividing roles clearly and communicating regularly, we were able to overcome these challenges and deliver a working solution on time.
