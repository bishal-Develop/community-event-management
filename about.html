Work Distribution & Roles

Our group divided the application into clear functional areas to streamline development. Alex focused on building a consistent layout using HTML and Bootstrap, including the fixed navigation bar, responsive two-column layout, and reusable card components. Priya implemented the event data pipeline using JSON, wrote the asynchronous fetching logic, and added client-side sorting, searching, category filtering, and pagination to keep the interface responsive and maintainable.

William implemented the lightweight authentication system using browser localStorage, including sign-up, login, hashed password storage with the Web Crypto API (SHA-256), session handling, route guarding, and logout. Sofia prepared the event dataset and asset references, mapped images from Unsplash to topics, verified Google Maps embeds, and ensured that event descriptions followed a clear, factual style suitable for listings and a detailed event details page.

Project Highlights

A key highlight is the dynamic events listing powered by a simple JSON data source and client-side rendering. The interface supports fast text search, category-based filtering, date range filtering, and intuitive pagination (five items per page) without requiring a backend. We also added “Read more” deep links from each card to a dedicated event details page using a query parameter (?id=) to fetch and display the correct record.

Another highlight is the approachable, Bootstrap-based design that remains responsive across breakpoints. We adopted a uniform card layout with image cropping via object-fit for clean visual consistency. The authentication flow demonstrates a fully front-end approach to storing user sign-ups and sessions, including hashed passwords, while keeping the code simple and easy to reason about for learning purposes.

Challenges Faced

One challenge was coordinating state between search, filters, sorting, and pagination. We addressed this by centralizing the render cycle: all user actions flow into a single render function that applies sorting and then paginates the filtered dataset. Another challenge was balancing a fixed viewport layout (to keep the pagination visible) with a scrollable events area; we resolved this using a constrained vh-based height and a sticky pagination bar inside the scroll container.

Implementing a front-end–only authentication system also required careful handling of password hashing and session state. We used the Web Crypto API for hashing and isolated session data to a dedicated localStorage key. Since this is a learning project without a server, we documented the limitations (e.g., client-side security constraints) and ensured the UX communicates expected behavior clearly.
